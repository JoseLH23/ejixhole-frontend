# Política de seguridad

## Versión soportada

La rama `main` representa la versión soportada del proyecto.

## Comunicación privada

Los problemas de seguridad deben comunicarse de forma privada mediante la opción **Report a vulnerability** de GitHub, cuando esté disponible, o directamente con `@JoseLH23`.

No incluyas información sensible en issues públicos. El reporte debe indicar el componente afectado, el commit o versión, el impacto esperado y los pasos mínimos necesarios para comprender el problema.

Objetivos de atención:

- confirmación de recepción en 5 días hábiles;
- evaluación inicial en 10 días hábiles;
- corrección priorizada según severidad y riesgo operacional.

## Reglas

- Las verificaciones sobre ambientes reales requieren autorización previa.
- No deben usarse datos reales de clientes para reproducir problemas.
- Todo valor confidencial expuesto debe revocarse y reemplazarse, no solamente eliminarse del repositorio.
